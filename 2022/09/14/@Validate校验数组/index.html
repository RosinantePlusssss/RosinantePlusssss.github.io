今天使用@validate校验参数List集合发现不生效。

@RequestMapping(value = "/test")
public ResponseEntity<String> test(@Validated @RequestBody List<TestDto> testDto, BindingResult br) {
    if(br.hasErrors()){
        String field = br.getFieldError().getField();
        System.out.println(field);
    }
    return new ResponseEntity<String>("test1121",
            HttpStatus.OK);
}

以上代码在校验失败，因为@Validated只能验证单个实体类，对List集合则不生效。百度一下发现可以对List进行封装一下使用@Valid注解即可。

public class ValidList<E> implements List<E> {

    @Valid
    private List<E> list = new ArrayList<>();

    public List<E> getList() {
        return list;
    }

    public void setList(List<E> list) {
        this.list = list;
    }

   ......

}

定义一个新的List类就可以了，实体类采用原先的校验注解即可，修改后的controller为：

@RequestMapping(value = "/test")
public ResponseEntity<String> test(@Validated @RequestBody ValidList<TestDto> testDto, BindingResult br) {
    if(br.hasErrors()){
        String field = br.getFieldError().getField();
        System.out.println(field);
    }
    return new ResponseEntity<String>("test1121",
            HttpStatus.OK);
}
