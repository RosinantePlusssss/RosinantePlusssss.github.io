哪些情况下数据库索引会失效

发表于 2022-02-18 更新于 2022-03-14 分类于 SQL
本文字数： 873 阅读时长 ≈ 1 分钟

索引失效情况。

今天到看一个B站up总结关于索引失效的问题为：模型数运空最快。

所以我打算用mysql进行验证一下是否正确。

模 ： 模糊查询的意思。like的模糊查询以%开头，索引失效。

SELECT * from test_user p where p.`name` like ‘%1’;

型 ：代表数据类型。类型错误，如字段类型为varchar，where条件用number，索引也会失效。

SELECT * from test_user p where p.area_id =1;

数 ：是函数的意思。对索引的字段使用内部函数，索引也会失效。这种情况下应该建立基于函数的索引。

SELECT * from test_user p where concat(p.area_id,’1’) = ‘21’;

空 ：是Null的意思。索引不存储空值，如果不限制索引列是not null，数据库会认为索引列有可能存在空值，所以不会按照索引进行计算。

视频中说is null没有走索引，但是实际测试中有走索引。
SELECT * from test_user p where p.area_id is null;（有走索引与结论不符。被认为索引失效的可能是因为mysql因为回表范围很大所以走全表扫描。）

视频作者建议大家这设计字段的时候，如果没有必要的要求必须为NULL，那么最好给个默认值空字符串，这可以解决很多后续的麻烦（切记）。

运 ：是运算的意思。对索引列进行（+，-，*，/，!, ！=, <>）等运算，会导致索引失效。

SELECT * from test_user p where p.equipment_id != 2;

虽然测试结果是不走索引，但是实际实况和上面是一样的，因为回表范围大走索引的时间更长所以才采用全表扫描。

最 ：是最左原则。在复合索引中索引列的顺序至关重要。如果不是按照索引的最左列开始查找，则无法使用索引。

快 ：全表扫描更快的意思。如果数据库预计使用全表扫描要比使用索引快，则不使用索引。
