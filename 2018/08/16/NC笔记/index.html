  57. 关于业务流中单据函数问题，在单据动作执行的时候就会触发单据函数，不管是不是最终执行审批都会触发，导致存在审批流的时候会多次触发单据函数，所以要加单据状态判断。

  58. 开发环境后台任务调试

  59. 关于流程定义的，PFBusiAction中会连续进行驱动，会按优先级继续进行下一个驱动，哪怕上一个驱动不成功，比如A审批驱动B的保存和审核，保存约束false也会进行下一个审批。在转换规则后事件中如果出现预计的错误可以返回null就行了。

  60. 关于单据模板的，有模板设置和模板分配，模板加载的顺序为：

    1. 分配到用户的模板->分配到用户组的模板->节点的默认模板->系统的单据模板
    2. 关于模板默认问题，查询pub_systemplate条件pk_corp=’00016910000000000LNJ’ and funnode=’THSA01’ and tempstyle=0 and NVL(nodekey,’‘)=’‘;之后根据策略匹配最高优先级。nodekey是功能节点默认模板设置中的节点标识。

  61. 关于工作流回退问题，回退当前节点判断依据pub_workflownote （网报流程历史）的处理时间、发送时间、ts倒序来倒退。如果三个时间都一样可能会出现当前动作取错导致报已有后续活动的错误。

报错

  1. 单据修改和复制导致单据最上面那个销售组织为空
    解决：改变元数据，将原先组织改为销售组织的

  2. 导入包正确但是报错
    解决：
    1. project-》clean
    2. 内存溢出。MDE 更新包路径
    3. 父类出错

  3. 保存时无效字符
    原因：BillCodeCheckRule单据号报错，没有单据号

  4. 保存时循环报错
    原因:新增属性的元数据和实体VO不一致，如一个动态一个静态 就会循环报错

  5. 新增按钮XML转化为JAVA报错

    1. 报错为空：可能是因为code没有注入
      解决方案：构造函数putValue(INCAction.CODE, “FlowreCordQuery”);
    2. 报错/或者无反应没有生成，解决方案：重启IDE

  6. Exception occurred during event dispathching: Object is not an instance of declaring class
    原因:找不到接口实现类,

    1. upm没有写好或者位置没有放对
    2. 开发环境ejb问题
      解决方案：
    3. 检查ump
    4. 开发环境下删除nchome的ejb文件夹

  7. nc.bs.framework.ConnecttorIOException/java.io.EOFException
    原因：元数据新增字段，没有勾选动态属性的时候VO没有添加普通字段。
    解决方案：
    动态属性 勾选后生成的VO中不显式声明属性字段，值按map<K,V>的格式保存，规范要求勾选

    1. 勾选动态属性。
    2. 不勾选就要VO添加普通参数字段。
